# Linux & Networking Interview Questions

1. Day-to-day Linux performance troubleshooting — What is your general troubleshooting approach?
2. CPU is idle but load average is high — What do you check, and what is the difference between CPU utilization and load average?
3. How do you dig deeper into an IO bottleneck? Why are processes queued?
4. How do you check disk usage in Linux?
5. How do you check disk reads/writes per second?
6. What are other reasons for high disk read/write activity besides disk space issues?
7. Walk me through what happens when you type `google.com` into a browser.
8. What are `dig` and `nslookup` used for?
9. What is the purpose of `/etc/hosts`?
10. What is the purpose of `/etc/resolv.conf`?
11. If a ping to a server fails, does that mean the server is down?
12. Host A cannot reach Host B. How would you troubleshoot the issue?
13. Why does a browser show an "Insecure Connection" warning?
14. On which ports does SSL/TLS operate?
15. When would you use an Application Load Balancer (ALB) vs a Network Load Balancer (NLB)?
16. Which load balancer would you recommend for a database workload, and why?
17. Explain the difference between RAM and Swap.
18. What is `vm.swappiness`?
19. Is high swap usage good or bad? Why?
20. What triggers an Out of Memory (OOM) condition?
21. What happens when the Linux OOM Killer is invoked?
22. Explain a recent production incident (RCA) that you handled.
23. How would you write a shell script (or automation) to check or update packages across 50 servers?
24. Tell me about a time you worked with a difficult or uncooperative teammate.
25. If a teammate's mistake caused a production outage, how would you explain it to your manager?
26. A NIC is not delivering the expected bandwidth/throughput. How would you troubleshoot it?
