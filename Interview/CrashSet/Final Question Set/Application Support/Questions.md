# 🖥️ Application Support Interview Questions

1. Describe your current production environment.

2. Tell us about your current project.

3. Can you talk about some of the projects you've worked on recently?

4. Can you talk about your last project?

5. What project did you enjoy working on the most?

6. Can you explain specifically what you worked on in that project?

7. Describe your production environment.

8. Describe a major production incident you handled.

9. Tell us about your most challenging production issue.

10. How do you troubleshoot an application outage?

11. Application is slow—how do you investigate?

12. Describe a situation where infrastructure was healthy but the application was failing. How did you identify the root cause?

13. What kind of web application was running there?

14. Technology-wise, what components did the application have?

15. How did you collect application metrics?

16. What tools did you use to search and analyze logs?

17. How was the Java application deployed?

18. Which application server did you use?

19. What experience do you have with AWS RDS and Aurora?

20. Have you performed database updates or migrations?

21. Have you worked in a small company where you handled multiple responsibilities instead of having dedicated database, network, and infrastructure teams?

22. Have you worked with InfluxDB?

23. What did you use InfluxDB for?

24. Can you explain the architecture of that application?

25. Why was there an EC2 instance in the request path?

26. What did you see in the VPC Flow Logs that helped you identify the issue?

27. What was the error or event in the VPC Flow Logs?

28. If it was a security group issue, why was the application working intermittently instead of failing all the time?

29. Walk me through a major problem you solved in your organization.

30. How did you become aware of the problem?

31. What information did you gather?

32. What information was missing?

33. How did you fill those gaps?

34. Did you perform a retrospective after the issue was resolved?

35. What did you learn from the incident?
