# AWS Production Issue - Deployment Failure (TCS)

## Situation

During a production deployment in TCS, the application deployment failed, and the new application version was not available to users.

## Task

My responsibility was to identify the reason for the deployment failure, restore the application availability, and support the successful deployment.

## Action

I first checked the deployment logs and application logs to understand where the deployment was failing.

I verified the server health, configuration changes, and required dependencies to identify the issue.

During the investigation, I found that an incorrect configuration value was causing the deployment failure.

I coordinated with the application team, corrected the configuration, and redeployed the application after validation.

After the deployment was completed, I monitored the application health and checked the logs to confirm that the application was working properly.

## Result

The deployment was completed successfully, and the application was restored without further issues.

As a preventive measure, we improved the deployment validation checklist to verify configuration changes before future releases.
