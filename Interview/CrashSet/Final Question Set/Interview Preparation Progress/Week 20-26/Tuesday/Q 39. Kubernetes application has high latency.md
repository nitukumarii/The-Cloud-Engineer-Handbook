# Kubernetes Application is Experiencing Latency

If a Kubernetes application is experiencing latency, I will first start by understanding the impact of the issue, like how many users are being affected and which services are actually slow.

I will first go through Dynatrace to understand the application performance metrics such as transaction rate, request rate, response time, latency, throughput, and error rate. For the exact error messages and application logs, I will use Splunk.

Since the application is running on Kubernetes, I will start with the Kubernetes health checks. I will check the pod health using `kubectl get pods`, node health using `kubectl get nodes`, and resource utilization using `kubectl top pods` and `kubectl top nodes`. I will also use `kubectl describe pod <pod-name>` and `kubectl get events` to see if there are any pod failures, restart issues, scheduling issues, or liveness/readiness probe failures.

If the Kubernetes resources look healthy but I still suspect an infrastructure issue, I will identify the node where the affected pod is running and log in to that particular node instead of checking every node.

At the Linux level, I will check CPU utilization using `top`, memory using `free -m`, disk utilization using `df -h`, and system logs using `journalctl`. I will also use `iostat` to check disk I/O and I/O wait, and `vmstat` to get an overall view of CPU, memory, and system performance.

If the infrastructure is healthy, I will move to the application layer to check for slow database queries, third-party dependency delays, network issues, or any recent deployment or configuration changes that could have introduced the latency.

Based on my findings, I will identify the root cause and apply the appropriate fix.

Once the issue is resolved, I will validate the application by monitoring the request rate, response time, latency, throughput, and error rate in Dynatrace, and verify through Splunk logs that there are no new errors.

Finally, I will document the RCA and implement preventive actions if required to avoid similar incidents in the future.
