# Change Management vs Problem Management

**Change Management:**
Change management is the process of planning, reviewing, approving, and implementing any changes to the production environment in a controlled way to reduce risk.

Examples:
- Application deployment
- Configuration changes
- Infrastructure changes
- Security updates

**Problem Management:**
Problem management focuses on identifying the root cause of recurring incidents after the immediate issue is resolved. The goal is to prevent the same issue from happening again.

Examples:
- Performing RCA after a major incident
- Identifying the root cause
- Implementing permanent fixes
