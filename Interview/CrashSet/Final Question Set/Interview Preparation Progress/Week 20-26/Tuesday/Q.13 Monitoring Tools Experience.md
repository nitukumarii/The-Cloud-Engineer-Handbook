
### Monitoring Tools Experience - Splunk, Dynatrace, CloudWatch, Grafana

I have good exposure to observability and monitoring tools across my projects.

I started my career working with AWS CloudWatch for infrastructure monitoring, where I monitored resources such as EC2 health, CPU utilization, memory, disk, 
network metrics, and application alerts.

During my time at Fidelity, while working with the performance testing and DevOps teams, I worked with Grafana dashboards. I configured JMeter Backend Listener to collect
performance test metrics and visualize them through Grafana dashboards. 
I created dashboards to monitor key performance metrics such as error rate, 95th percentile response time, throughput, and latency.

In my recent SRE role, I worked extensively with Splunk and Dynatrace for production monitoring. I used Splunk for log analysis and 
troubleshooting application errors, while Dynatrace helped monitor application performance, transaction flow, response time, dependencies, and overall application health.
