# Explain Ansible Playbook

An Ansible playbook is a YAML file that defines the automation tasks to be performed on target hosts.

A playbook contains one or more plays, and each play defines the target hosts and the tasks that need to be executed.

Each task uses Ansible modules to perform specific actions, such as installing packages, managing services, updating configurations, or restarting applications.

For example, a playbook can be used to patch Linux servers, install Nginx, update configuration files, and restart services automatically.
