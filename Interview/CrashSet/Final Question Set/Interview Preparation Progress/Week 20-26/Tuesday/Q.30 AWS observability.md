# AWS Observability Services

In AWS, we use different services for monitoring and observability.

- **CloudWatch** is used for collecting infrastructure metrics, application logs, alarms, and dashboards. We monitor CPU, memory, disk, application errors, and performance metrics.

- **CloudTrail** is used for auditing AWS API calls and tracking changes made in the AWS environment.

- **AWS X-Ray** can be used for distributed tracing to understand application request flow and identify performance issues.

- **AWS Security Hub and GuardDuty** are used for security monitoring and detecting potential threats.

For cost optimization, we use services like **AWS Cost Explorer and Compute Optimizer** to analyze resource usage and optimize costs.
