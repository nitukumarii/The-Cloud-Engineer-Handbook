# SRE Toil

SRE toil refers to repetitive, manual, and operational tasks that consume engineers' time but do not provide long-term value or improve the system.

Examples of toil:
- Manual deployments
- Repeated incident checks
- Manual server configuration
- Manually restarting services
- Repeated log analysis

SRE teams try to reduce toil by automating these tasks using tools like Python, Bash, Ansible, Terraform, and CI/CD pipelines.
