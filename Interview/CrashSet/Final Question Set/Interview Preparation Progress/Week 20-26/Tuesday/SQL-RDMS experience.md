# SQL / RDBMS Experience Explanation

I have worked with relational databases mainly from a production support and SRE perspective rather than as a dedicated database administrator.

In my previous roles, I supported applications that depended on databases, where my responsibilities included monitoring database connectivity, 
troubleshooting application database-related issues, checking logs for database errors, and coordinating with database teams during production incidents.

I have experience with basic SQL queries to validate data, check transaction status, and support troubleshooting activities. For example, 
during production issues, I have checked whether transactions were successfully inserted or updated, identified failed transactions, and verified application behavior from the database side.

I have also worked with AWS database services like RDS at an infrastructure level, including understanding connectivity, security groups, availability, backups, and monitoring.

For database-related incidents, my approach is to check application logs first, validate database connectivity, review error messages, check resource
utilization, and coordinate with the DBA team if deeper database investigation is required.
