- **User CPU (us):** Percentage of CPU time spent executing user-space application processes; a high value usually indicates that applications are actively consuming CPU.

- **System CPU (sy):** Percentage of CPU time spent executing kernel-space operations such as system calls, interrupt handling, process scheduling, and device drivers.

- **Idle CPU (id):** Percentage of CPU time during which the processor is idle and not executing any tasks, indicating available CPU capacity.

- **I/O Wait (wa):** Percentage of CPU time spent waiting for disk or storage I/O operations to complete, often indicating a storage bottleneck rather than a CPU problem.

- **Steal Time (st):** Percentage of CPU time a virtual machine waits because the hypervisor is allocating the physical CPU to other virtual machines, indicating host-level CPU contention.

- **Load Average:** The average number of processes that are running or waiting for CPU or uninterruptible I/O over the last 1, 5, and 15 minutes, indicating the overall system workload.
